웹 서비스 인증(UI) 화면을, “AI-Music-Mood-Player” 스타일 가이드에 맞춰 디자인해줘.

요구사항:

1) 공통 스타일 가이드
전체적으로 어두운 테마 (다크 모드)
미니멀 & 모던 느낌: 심플한 카드/패널, 넉넉한 여백, 플랫(flat)한 UI
폰트/텍스트: 깔끔한 산세리프, 부드러운 크기와 간격
버튼 & 입력 필드: 부드러운 모서리, 약간의 그림자 또는 음영, hover / focus 시 색상 변화
색상 팔레트: 어두운 배경 + 포인트 컬러(예: 네온 블루, 민트, 밝은 파랑 계열)
아이콘 / 심볼: 심플하고 선(line)-스타일 아이콘, 필수 요소만

2) 화면 구성

   a) 로그인 화면
배경: 어두운 톤 (예: #121212 또는 #1e1e1e)
로그인 카드: 반투명 / 반광택 느낌 (glassmorphism or dark card)
입력 필드: 이메일, 비밀번호 — placeholder + 라벨 포함
버튼: “로그인” — 포인트 컬러, hover 시 조금 더 밝게
링크: “회원가입”, “비밀번호 찾기” — 작은 텍스트 버튼
로고 / 제목 영역: 상단에 로고 또는 서비스 이름 + 부제, 세련된 타이포

   b) 회원가입 화면
로그인 화면과 동일한 스타일 기반
입력 필드: 이름, 이메일, 비밀번호, 비밀번호 확인
입력 필드 간 여백 넉넉히
버튼: “회원가입” — disabled 상태 (동의 체크 전)와 활성화 상태 구분
아래에 개인정보 동의 체크박스 섹션

   c) 개인정보 동의 체크 박스 영역
카드 또는 패널 형태 (배경이 조금 더 투명하거나 그림자 포함)
“전체 동의” 체크박스 + 개별 동의 항목들
[필수] 서비스 이용약관
[필수] 개인정보 수집/이용
[선택] 마케팅 정보 수신
각 항목 오른쪽에 “자세히 보기” - 아이콘 또는 작은 링크 (ex. chevron icon)
“전체 동의”가 선택되면 모든 개별 항목 자동 선택 / 동의 해제 시 개별 컨트롤 가능
동의되지 않은 상태이면 “회원가입” 버튼은 비활성화

3) UI 흐름
로그인 화면 ↔ 회원가입 화면 ←> 개인정보 동의 체크
(옵션) 약관/개인정보 내용 보기 모달 팝업 스타일: same dark theme + 반투명 overlay + 카드 레이아웃
넣어야할것 : 로그인후 로그아웃 버튼을 활성화시키고, 로그인한 프로필을 유지(우측 상단 프로필 버튼을 누르면 사용자가 로그인한 정보를 볼수 있도록)할 수 있도록하기, 로그아웃시 첫 로그인, 회원가입페이지로 전환 되게 끔 바꿔줘


로그인후 메인화면 진입 시 로그인/회원가입버튼, Sporify 로그아웃 버튼은 없에주고 프로필, 로그아웃, Spotify 로그인 버튼만 활성화 시켜줘.

[B] 예정 

AI-Music-Mood-Player 스타일에 맞춰 계정 기반 기능 UI 세트를 디자인해줘.
전체 테마는 다크 톤, 네온 포인트 컬러, 미니멀한 카드 레이아웃을 유지해줘.

포함해야 하는 화면들:

1) 마이페이지(My Page)
   - 다크 톤 배경에 둥근 카드 형태 구성
   - 사용자 프로필 카드(이름, 이메일, 프로필 아이콘)
   - 설정 섹션:
       * “사용 기록 저장 활성화” 토글 스위치
       * “계정 정보 수정” 버튼
       * “비밀번호 변경” 버튼(선택)
       * “로그아웃” 버튼(포인트 컬러)
   - 정보가 깔끔하게 정렬된 미니멀 UI
   - 포인트 컬러는 네온 블루 또는 민트 사용

2) 사용 기록 저장 기능 안내 모듈
   - 마이페이지 또는 온보딩에서 노출할 안내 카드:
       * “활동 기록을 저장하여 나중에 다시 볼 수 있어요”
       * 작은 아이콘(시간/리스트/북마크 등)
       * “기록 저장 활성화” 토글 스위치
   - 반투명 카드 또는 약한 글래스모피즘 스타일로 표현

3) 사용 기록 조회 화면(User Activity History)
   - 최신 기록이 상단에 표시되는 리스트 카드 UI
   - 각 기록 아이템 구성:
       * 날짜/시간
       * 활동 설명 (예: “Mood: Chill 선택”, “음악 재생: Dreamy Track”)
       * 작은 카테고리 아이콘
   - 상단 필터:
       * 전체보기 / 무드 선택 / 음악 재생 / 설정 변경
   - 스크롤 가능한 리스트 형태
   - 기록이 없을 때:
       * 빈 상태 일러스트 또는 “아직 기록이 없습니다”

4) 활동 상세 보기 화면(Activity Detail)
   - 기록 하나를 눌렀을 때 나타나는 상세 카드
   - 구성 요소:
       * 활동 제목
       * 날짜/시간
       * 관련 이미지/아이콘
       * 상세 정보(예: 선택한 무드, 재생된 음악 제목, 지속 시간 등)
   - 닫기 버튼 또는 뒤로가기 버튼 포함

6) 전체 디자인 스타일
   - 배경: #0F0F0F–#171717 범위의 다크 모노톤
   - 카드: 어두운 반투명 패널 또는 약한 블러
   - 포인트 컬러: 네온 블루(#4EE0FF), 민트(#39E7C3) 계열
   - 버튼: 라운드 + 글로우 효과 약하게
   - 아이콘: 라인 아이콘(화이트/블루 계열)
   - 폰트: 깔끔한 산세리프 + 넉넉한 자간
   - 전체적으로 모던하고 세련된 느낌 유지

마이페이지를 프로필 버튼을 놀렀을때 메인페이지에 노출시키지 말고 카드형식으로 프로필 페이지 내에 띄워줘

AI-Music-Mood-Player 스타일을 기반으로, 개인 음악 감상 앱을 “공유 가능한 감정/분위기 커뮤니티”로 확장한 UI 세트를 디자인해줘.

전체 테마:
- 다크 톤 + 네온 포인트 컬러 기반
- 미니멀한 카드 레이아웃
- 부드러운 애니메이션과 모션 효과를 전제로 한 구성
- 모바일/태블릿/데스크톱까지 반응형 레이아웃을 고려해서 디자인

포함해야 하는 화면들:

1) 메인 피드 화면 (Mood 커뮤니티 피드)
   - 사용자가 선택한 무드와 플레이리스트, 감정 상태를 공유하는 카드 리스트 형식
   - 각 카드 구성:
       * 프로필 아이콘 + 닉네임
       * 날짜/시간
       * 무드 태그 (예: #Chill, #Focus, #Happy)
       * 간단한 코멘트 텍스트 (예: “이 날은 이런 기분이었어요”)
       * “현재 재생 중이었던 트랙/플레이리스트” 미니 정보
       * 좋아요(하트), 댓글 아이콘, 북마크/즐겨찾기 아이콘
       * 공유 버튼 (카드 공유 링크 생성용)
   - 상단에 필터:
       * 전체 / 내 무드 / 팔로잉 / 인기 무드
   - 카드에 hover 또는 탭 시 살짝 확대·그림자 강화되는 모션을 고려해서 디자인

2) 무드 선택 + 공유 카드 생성 화면
   - 사용자가 오늘의 무드를 선택하고, 코멘트와 함께 공유 카드를 만드는 화면
   - 구성 요소:
       * 무드 선택 버튼들 (Chill, Focus, Happy, Sad 등) — 선택 시 카드 색/조명이 살짝 변함
       * 텍스트 입력 영역: “오늘의 기분/상황을 적어보세요”
       * 현재 듣고 있는 음악/플레이리스트를 추가하는 옵션
       * “공유 카드 미리보기” 영역 — 실제 피드에 올라가는 카드 디자인을 실시간으로 보여줌
       * “공유하기” 버튼
   - 무드 버튼 클릭 시:
       * 버튼이 부드럽게 튕기듯 반응
       * 배경 그라데이션이나 주변 글로우가 살짝 바뀌는 애니메이션을 상상할 수 있게 디자인

3) 플레이리스트 공유 상세 화면
   - 특정 공유 카드나 플레이리스트를 눌렀을 때 보이는 상세 페이지
   - 구성 요소:
       * 공유한 사용자의 프로필/닉네임
       * 선택된 무드 태그
       * 코멘트 내용
       * 플레이리스트 트랙 리스트 (곡 제목, 아티스트, 재생 아이콘 등)
       * 좋아요 수, 댓글 수, 북마크 수 표시
       * “친구에게 공유하기” 버튼 (링크 복사 또는 공유 옵션 아이콘)
   - 뷰 전체가 카드 느낌이 나도록 다크 + 네온 스타일로 구성

4) 알림 센터 화면 (Notifications)
   - 상단에 “알림(Notification)” 제목
   - 알림 리스트:
       * 예: “OOO님이 당신의 무드 카드에 좋아요를 눌렀습니다.”
       * 예: “새 댓글이 달렸어요.”
       * 예: “팔로우한 사용자가 새로운 무드/플레이리스트를 공유했습니다.”
   - 각 알림 아이템은 아이콘 + 텍스트 + 시간 정보로 구성
   - 읽지 않은 알림은 네온 포인트 컬러로 강조된 상태 표시
   - 알림 화면은 모바일 기준 모달/패널, 데스크톱 기준 오른쪽 사이드 패널 느낌으로 디자인

5) 좋아요 / 즐겨찾기 / 북마크 / 댓글 기능 UI
   - 카드/리스트 아이템 공통 액션 바 디자인:
       * 하트 아이콘(좋아요)
       * 북마크 아이콘(즐겨찾기/저장)
       * 말풍선 아이콘(댓글)
       * 공유 아이콘
   - 각 아이콘은 클릭/탭 시 색이 네온 컬러로 채워지면서 부드러운 확대 애니메이션이 들어간 것처럼 디자인
   - 댓글 영역:
       * 상단에 기존 댓글 리스트
       * 하단에 댓글 입력 필드 + 전송 버튼
       * 다크 톤의 말풍선 스타일로 정리된 레이아웃

6) 테마 커스터마이징 화면 (Theme Settings)
   - 사용자 설정(Settings) 내의 테마 변경 화면
   - 선택 가능한 테마 예시:
       * “Dark + Neon” (기본 테마)
       * “Light + Pastel”
       * “Neon Retro”
       * “Minimal Black”
   - 각 테마를 보여주는 미니 프리뷰 카드들을 가로 스크롤 또는 그리드로 배치
   - 현재 선택된 테마는 네온 테두리 또는 체크 아이콘으로 강조
   - 하단에 “테마 적용” 버튼

7) 반응형 레이아웃 가이드
   - 모바일, 태블릿, 데스크톱 3가지 브레이크포인트 기준으로 주요 화면(메인 피드, 카드 상세, 알림센터, 테마 설정)의 레이아웃 예시를 보여줘.
   - 모바일:
       * 세로 스크롤 중심, 카드 하나씩 꽉 차게
   - 태블릿:
       * 2열 카드 그리드 또는 사이드바 + 콘텐츠 구조
   - 데스크톱:
       * 왼쪽 사이드 내비게이션(홈, 커뮤니티, 알림, 설정)
       * 오른쪽에 피드/상세/설정 화면이 나타나는 형태

8) 애니메이션/모션 표현 가이드
   - 실제 프로토타입에서 사용할 수 있도록, 다음 상호작용에 대해 애니메이션 전제를 반영한 UI로 디자인:
       * 무드 버튼 클릭 시: 버튼이 살짝 커졌다가 돌아오는 마이크로 인터랙션
       * 좋아요/북마크 클릭 시: 아이콘이 채워지면서 짧은 글로우 효과
       * 페이지 전환 시: 페이드 + 슬라이드 조합(예: 오른쪽에서 왼쪽으로 슬라이드 인)
   - 실제 모션을 구현하지는 않아도, 어떤 모션이 들어갈지 상상 가능한 레이아웃과 상태 디자인을 해줘.

전체 디자인 스타일:
- 배경: #0F0F0F ~ #171717 다크 모노톤
- 카드: 어두운 반투명 + 약한 글래스모피즘 효과
- 포인트 컬러: 네온 블루, 민트, 핑크 등(무드에 따라 적당히 활용)
- 폰트: 심플한 산세리프, 넉넉한 자간과 줄 간격
- 전체적으로 “음악 + 감정 + 커뮤니티”가 느껴지는 모던하고 세련된 UI로 만들어줘.

위 구성 요소들을 모두 포함해서,
“무드 선택 및 공유 → 커뮤니티 피드 → 좋아요/북마크/댓글 상호작용 → 알림 → 테마 설정”
까지의 흐름이 한눈에 보이도록 하나의 파일 안에 여러 프레임으로 배치해줘.
